设计要求与硬件配额

本工程面向 TEC-8 实验箱 以及 EPM7128SLC84-15 CPLD 芯片。由于该芯片的逻辑容量极为受限（仅 128 个宏单元），因此对代码设计提出了苛刻的要求。

⚠ 关键硬件物理约束
1. LG1 秒个位驱动必须直译码（LG1-D0..D6，Pin 58..64），LG2..LG6 采用 4-bit BCD 驱动输出。
2. 扬声器驱动与直译码小数点（LG1-D7）复用物理引脚 PIN_52。设计中必须小心处理蜂鸣时钟，不可误点亮数码管。
  • 全同步设计：使用 1KHz 的 CP2 时钟作为主工作时域，将按键和秒脉冲同步化，杜绝了电路板上的竞争冒险。
  • 零多余资源：将计数寄存器直接定义为六个 BCD 码，避开了复杂的二进制-BCD码译码电路，省下了宝贵的宏单元。
  • 防误触逻辑：闹钟设置状态 K2=1 时，QD 按键被屏蔽，避免调闹钟时误触发系统开始/暂停计时。

1 KHz

CP2 主同步工作频率，每 1ms 重新评估所有输入脚状态，进行数字锁存。

平台物理配额

戴着镣铐跳舞

EPM7128SLC84 的宏单元资源非常紧缺。任何多余的逻辑都将导致综合器无法布线：

  • 当前使用度：120 / 128 个宏单元 (占比达 93.8%)。
  • 仅剩空间：8个宏单元，后续功能扩展需极度小心。
  • 优化手段：全系统使用 BCD 寄存器，规避昂贵的二进制-十进制译码逻辑，释放布线压力。
避雷忠告：切勿在此 CPLD 设备上盲目引入倒计时、反向校时或者长按加速逻辑，否则编译器会因宏单元溢出而报错！
系统架构

心跳与逻辑分布

为了让整个系统足够精简，我们仅部署两个外部时钟域，极大地降低了电磁干扰：

  • CP2 时钟 (1KHz)：主逻辑同步与按键滤波。扬声器在此翻转得到 ~500Hz 蜂鸣音色。
  • CP3 脉冲 (1Hz)：走时基准脉冲。在同步电路中作为使能标志而非直接作为触发器时钟。
  • PIN_52 双重化：扬声器与 LG1 数码管共用引脚，设计中引入软件静默隔离。
wire cp3_rise = (cp3_sync[2:1] == 2'b01);
wire qd_rise = (qd_sync[2:1] == 2'b01);

// 仅使用一个时钟边沿更新全局状态
always @(posedge cp2) begin
  cp3_sync <= {cp3_sync[1:0], cp3};
  qd_sync <= {qd_sync[1:0], qd};
end
时序优化

告别异步乱跳

第一代版本采用 `posedge cp3` 和 `posedge pulse` 的多时钟触发器。这引发了严重的硬件毛刺：

  • 痛点：直接使用机械开关脉冲驱动计数器时钟，会导致极严重的接触抖动，使数码管数字乱跳。
  • 方案：本版本全部修正为 CP2 主时钟同步采样。CP3 及 Pulse 仅作为判断条件。
  • 收益：波序平直干净，数码管无任何乱闪，成功编译通过 Quartus 时序验证。

功能扩展

多功能闹钟与整点报时

高利用率并不影响功能的丰富性。时钟集成了完整的声响反馈系统：

  • HH:MM:SS 秒级闹钟：可调节到具体某秒。到点后以 1Hz 频率滴答短响，支持任意时刻按 QD 消音。
  • 整点短报时：K4 开启后，在运行进入整点瞬间短鸣 125ms，校时过程中不报时，防止噪音。
  • 安全互斥：K2 在调闹钟时屏蔽 QD，避免误改变主时钟的走停状态。

K3 闹钟控制

独立锁存闹钟，按 QD 进行消音时不会停止当前时间的计时运行。

K4 整点控制

检测低16位分和秒是否为全 0，触发 125ms 的快速时钟中断短鸣。

极简函数与结构设计

为了重用加法器并最大限度压缩生成的乘积项，我们将小时/分钟进位独立写成复用函数：

  • inc_hour_pair：执行 24 小时翻转以及十位/个位十进制加法。
  • inc_minute_pair：执行 60 分钟翻转并自动级联小时增量。
  • seg7_cc：秒个位共阴极数码管纯组合逻辑译码，消除内部多余分配器。
function [7:0] inc_hour_pair;
  input [7:0] current;
  begin
    h_tens = current[7:4]; h_ones = current[3:0];
    if (h_tens == 4'd2 && h_ones == 4'd3) begin
      inc_hour_pair = 8'h00;
    end else ...
  end
endfunction
